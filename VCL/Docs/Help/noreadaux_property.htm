NoReadAux property

  • Working with SFX zips
  • List
  • ZipFileName
  • Working with multi-part zips
Applies to

TZipMaster component

Declaration

property NoReadAux: Boolean;

Description

When an existing spanned zip file or SFX file is read, by setting ZipFileName and this property is false (the default), the component will try to read some parameters in the existing archive.

For spanned archives it will read the number scheme used and set the SpanOptions property accordingly.
For SFX files it will read the SFX properties and set the SFX properties accordingly.

SFX properties read are

  • SFXCaption
  • SFXDefaultDir
  • SFXCommandLine
  • SFXRegFailPath
  • SFXOptions
  • SFXOverWriteMode
  • SFXMessage

If NoReadAux is set to true these values are not read and the default values will be used.

Setting any of the above SpanOptions or SFX properties to a value other than the default value in the Object inspector NoReadAux will be read as true to avoid overwriting of your settings.
