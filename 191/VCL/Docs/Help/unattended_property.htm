Unattended property

  • ErrMessage
  • Verbose
  • Trace
Applies to

TZipMaster component

Declaration

property Unattended: Boolean;

Description

Unattended is an option to make sure you won't get ANY error and/or warning messages from the component or the DLL.

Default is that error or warning messages are given by the component/DLL which need user interaction.

With this property set to True ALL messages are suppressed. This also implies that you will not be visually informed of problems while executing your code.

NOTE: All messages are still available in the OnMessage event and the ErrCode and ErrMessage properties.

Example
 If you want to use this property to test for bad zipfiles look at the following code:        
          
 // Set Unattended to avoid technical errors from being displayed.        
 // You can still catch error messages in your OnMessage handler, if you want.        
 ZipMaster1.Unattended := True;        
          
 // The open occurs when you assign the filename        
 ZipMaster1.ZipFileName := a_File_Name;        
          
 // If the zipfile is bad, the open fails and the filename is set to an empty string.        
 If ZipMaster1.ZipFileName = '' then        
 begin        
   // Do something: (skip) (check the ErrCode property) (write to a log file)        
   ShowMessage( 'Error in Zipfile' + IntToStr( ZipMaster1.ErrCode) );        
 end else        
 begin        
   ZipMaster1.Unattended := False; // Not needed, depends on the next action        
   // Continue processing.           
 end;        
 // Turn on Unattended again in case there's another problem that we are not looking for.        
 ZipMaster1.Unattended := False;        
          
 NOTE: All messages are still available in the OnMessage event and the ErrCode and ErrMessage properties.
