Changes v1.9

New
  • Zip extensions version 4.50 -
    • Files larger than 4Gb supported
    • More than 64k files
  • Zip file names - can use UTF8 for storing international file names.
  • When writing zip files the format used is controlled by new WriteOptions
    • zeoAuto, use OEM unless extended characters, otherwise UPath or UTF8
    • zeoRaw, do not encode file names
    • zeoOEM, convert file names to OEM
    • zeoUTF8, store extended file names as UTF8 (Default)
    • zeoUPATH store extended file names using unique safe name and (Info-Zip) UTF8
  • When reading zip files the format expected is controlled by the new property Encoded
    • zeoAuto, unless flags/versions say otherwise, or it has UTF8 name in data, treat it as OEM (FAT) / Ansi (NTFS)
    • zeoNone, assume name is Ansi - convert to Ansi/Unicode
    • zeoOEM, assume name is OEM - convert to Ansi/Unicode
    • zeoUTF, assume name is UTF8 - convert to Ansi/Unicode
    • zeoUPATH, use UPATH if available
  • Pre-D2009 to prevent corruption of international names they can be kept as UTF8 - see property UseUTF8.
  • D2009 (and later) file names are converted to Unicode so no corruption should occur.
  • NTFS file stamps can be saved or restored - see AddNTFS and ExtrNTFS.
  • When adding new files to an existing archive the old file can be preserved - see AddVersion.
