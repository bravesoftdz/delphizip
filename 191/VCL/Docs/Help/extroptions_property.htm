ExtrOptions property

  • Extract
  • NoSkipping
Applies to

TZipMaster component

Declaration

property ExtrOptions: TZMExtrOpts;

type
  TZMExtrOpts = set of TZMExtrOptsEnum;

type
  TZMExtrOptsEnum = (ExtrDirNames, ExtrOverWrite, ExtrFreshen, ExtrUpdate, ExtrTest, ExtrForceDirs, ExtrNTFS);

ExtrDirNames
If True, extracts and recreates the relative pathname that may have been stored with each file.
Empty directories stored in the archive (if any) will also be recreated.
ExtrOverWrite
If True, overwrite pre-existing files during Extraction.
NOTE: -ReadOnly files will be overwritten too.
If a file will be overwritten also depends on the return value set in the OnExtractOverwrite event if present.
IMPORTANT: Files opened by other applications (Windows a.o.) with exclusive rights are not overwritten. In this case an OnSkipped event will follow.
ExtrFreshen
If True, extract newer files from archive (only for files that already exist).  Won't extract any file that isn't already present.
ExtrUpdate
If True, extract files that don't already exist and if ExtrOverWrite is also set to true, it will extract newer files from the archive.
ExtrTest
If True, only test the archive to see if the files could be successfully extracted.
Testing is done by extracting the files, but NOT writing the extracted data to the disk.
Only the CRC code of the files is used to determine if they are stored correctly.
To use this option, you will also need to define an Event handler for OnMessage.
ExtrForceDirs
If True, forces the existence of the destination directory.
ExtrNTFS
If True, restores file NTFS time stamps.
Description

Controls extracting files from zip archive
