A) How to copy each part to diskette afterwards:

1) Make the MV archive on harddisk. Your filenames will be, for example:

'Test001.zip', 'Test002.zip', 'Test003.zip'

2) Copy each of them to it's own diskette.

3) Rename each file on diskette to 'Test.zip'

4) Set the label of the diskettes to 'PKBACK# 001', 'PKBACK# 002' and 'PKBACK# 003'

( Note the space between # and the first digit! )

5) Start reading from the last diskette.

B) How to reassemble the pieces at the client side after sending the files over the internet, without using diskettes.

1) Start a DOS window and "CD" to the directory with Test???.zip

2) Concatenate them with:

"Copy /B Test001.zip+Test002.zip+Test003.zip Test.zip" or shorter:  "Copy /B Test???.zip Test.zip"

(Do not enter the quotes.)

3) Fix the internal directory structure with: PKZipFix Test.zip

Note: PKZipFix.EXE comes with pkzip v2.04g.

4) Rename the resulting file 'PKfixed.zip' to 'Test.zip' (This is now a normal zip file and should be 4 bytes smaller than the sum of the parts.

 

NOTE1: There is no time penalty making the archive directly on diskettes because the ZipMaster and ZipBuilder components internally use a temporary file on harddisk.

NOTE2: Harddisk spanning is not supported by PKZip and WinZip, so if you want to use files created this way with these programs you will have to use one of the above mentioned methods.
