DLL Loading and Unloading

DLL Loading and Unloading

This table shows you which methods require the DLL:

  • Add
  • AddFileToStream
  • AddStreamToStream
  • Extract
  • ExtractFileToStream
  • ExtractStreamToStream

 

All other methods do not need the DLL.

NOTE:  'Test' is a sub-option of Extract.

NOTE: If static binding of the DLL to the component is used then the DLL is always loaded or the application (or the IDE) will not load.

The following properties give you control over loading and unloading the DLL

  • DLLDirectory optionally specify where to look for the DLL.
  • DLL_Load Set true to load the DLL, false to unload - returns true if loaded
  • DLL_Path returns the path from which the dll was loaded
  • DLL_Build returns the DLL build
  • DLL_Version returns the DLL version

For compatibility with older programs the loading and unloading will be done automatically if your program does not do it.

This can, however, incur performance penalty because it will reload the DLL for each operation.
