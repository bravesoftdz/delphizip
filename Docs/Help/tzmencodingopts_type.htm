type
  TZMEncodingOpts = (zeoAuto, zeoNone, zeoOEM, zeoUTF8, zeoUPath);

EncodeAs values (writing)
zeoUPATHconvert to Ansi in header but have UTF8 proper name in data
zeoUTFconvert to UTF8
zeoOEMconvert to OEM
zeoNonestore 'as is' (Ansi on Windows)
'default' (zeoAuto) [ in order of preference ]
  1. is Ansi - use zeoNone
  2. can be converted to Ansi - use zeoUPath (unless comment also extended)
  3. use zeoUTF8

 

Encoded values (reading)
zeoUPATHuse UPATH if available
zeoUTFassume name is UTF8 - convert to Ansi/Unicode
zeoOEMassume name is OEM - convert to Ansi/Unicode
zeoNoneassume name is Ansi - convert to Ansi/Unicode
'default' (zeoAuto)unless flags/versions say otherwise, or it has UTF8 name in data, treat it as OEM (FAT) / Ansi (NTFS)
Description

This is also the type returned by the TZMDirEntry.Encoded property for that particular entry.
