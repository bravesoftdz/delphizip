General

ZipSniffer is an application that allows you to collect all message send by the TZipMaster component without the help of the application that is using the component.
The main purpose is to collect as much information as possible in case of an error condition, you can then use this info yourself to find the problem source or send it as part of an error report.
To collect the message you need to activate ZipSniffer before the application is launched, then set the required options in ZipSniffer and launch the application.
TZipMaster all ways searches for the presence of ZipSniffer it it is present it will find it and send its message to ZipSniffer using the Windows messages mechanism.

When ZipSniffer receives message it will either show them in a child window and/or store them in a buffer. All messages stored in the buffer can be saved in a log file. That log file can be reviewed in the ZipSniffer using another child window, or can be treated as any other text file.

The operation mode of ZipSniffer is controlled by the options set by :

  • The Tool Buttons
  • The Menu Items

The display grid has the following columns:

  • Tick : cpu tick count
  • Operation number : each operation has a different number
  • Source unit (where known)
  • Source line (where known)
  • Error code
  • message text
