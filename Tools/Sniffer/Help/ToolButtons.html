ToolButtons

To know what a tool button is for, go over it with the mouse cursor and a little help will appear and at the bottom left of the main form a hint will appear.

ZipSniffer has 13 Tool buttons in 4 groups on its main form from left to right :

  1. File Group
    • New : Creates a new child form, you can use this form as you like.
    • Open : Opens a file selection dialog, the purpose is to open one of the log files, then it creates a new child form and shows the content of the file in that form.
    • Save : Not yet implmented.
  2. Edit Group
    • Cut : Cuts the actual selection and saves it to the clipboard, make a selection in one of the child forms first.
    • Copy : Copies the actual selection and puts it on the clipboard, make a selection in one of the child forms first.
    • Paste : Pastes the clipboard content into the selected child form.
  3. Window Group
    • Cascade: cascades the child forms.
    • Tile Horizontally : arranges child forms so that they are all the same size, tiled horizontally.
    • Tile Vertically : arranges child forms so that they are all the same size, tiled vertically.
  4. Options Group
    • Watch : toggles between logging and watching the received messages, when in watch state the first message will create a new child form. In watch state the message are also send to the buffer and the log file can also be created.
      The actual status is show at the bottom of the main form in the second panel.
    • Trace : Determines how the Trace property in the TZipMaster component is used, the actual state of Trace is indicated in the 4th panel, it is a tri-state option:
      • Indeterminate (the default) : indicated by 'Trace ?' will use the Trace options as it is set in the application using TZipMaster.
      • Off : indicated by 'Trace off' will force the Trace property to off.
      • On : indicated by 'Trace On' will force the Trace property to on.
    • Verbose : Determines how the Verbose property in the TZipMaster component is used, the actual state of Verbose is indicated in the 3th panel, it is a tri-state option:
      • Indeterminate (the default) : indicated by 'Verbose ?' will use the Verbose options as it is set in the application using TZipMaster.
      • Off : indicated by 'Verbose off' will force the Verbose property to off.
      • On : indicated by 'Verbose On' will force the Verbose property to on.
    • Save Log : Creates a log file and puts the content of the message buffer in that file. The log file is named dzlogxxx.txt where xxx is a number from 001, if that file is already open then we use 002 and so on.
