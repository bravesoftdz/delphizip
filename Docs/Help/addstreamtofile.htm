Applies to

TZipMaster component

Declaration

function AddStreamToFile(const FileName: String; FileDate, FileAttr: Dword): Integer;

Results
  • Returns 0 on success otherwise a - Error Code.
  • Sets ErrCode and ErrMessage
Description

The AddStreamToFile method adds the specified stream to the zip archive.

Adds stream data, which should be in ZipStream, to the zip archiveZipFileName under the filename given in the parameter FileName. Also a FileDate (DOS filedate and time) and some FileAttributes can be set.

- If NO FileName is passed as parameter then the first in the list of FSpecArgs will be taken as FileName. Wildcards are not allowed!

- If NO FileDate is passed as parameter, the local system date and time will be used.

- FileAttr can be one or a logical combination of the following types:

FILE_ATTRIBUTE_ARCHIVE, FILE_ATTRIBUTE_HIDDEN, FILE_ATTRIBUTE_READONLY, FILE_ATTRIBUTE_SYSTEM.

If NO FileAttr is passed as parameter, it will be set to 0.

Example
 Example: // Add something as data to the ZipStream...        
 ZipMaster1.ZipStream.LoadFromFile( 'L:\SomeFile.dat' );        
 // *** Other possibilities ***        
 // ZipMaster1.ZipStream.LoadFromStream( aStream ); // Or use data from another stream.        
 // ZipMaster1.ZipStream.Write( aTextbuffer, NumberOfBytes ); // Or data from a character buffer.        
 ZipMaster1.AddStreamToFile( 'MyFilenameToAdd.dat' );        
 // *** Another possibility ***        
 // ZipMaster1.FSpecArgs.Clear;        
 // ZipMaster1.FSpecArgs.Append( 'MyFilenameToAdd.dat' );        
 // ZipMaster1.AddStreamToFile;        
 // *** Another possibility ***        
 // ZipMaster1.AddStreamToFile( 'MyFilenameToAdd.dat', 0, FILE_ATTRIBUTE_ARCHIVE );        
 // *** Another possibility ***        
 // ZipMaster1.AddStreamToFile( 'MyFilenameToAdd.dat', TDateTime( '6-4-00 14:32' ).FileDate );
