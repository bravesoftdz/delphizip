Filespec Switches

  optional arguments between []

  MUST BE in specified order

Add and extract

  [| OR >] filespec [ /switch [ /switch ...]] [<password]

  • |  stops recursing folders *add only
  • > forces recursion *add only

filespec the required filename

/switch optional switch of form /char=value (note / must follow space)

  • c: set compression (0..9) *add only
  • e: excludes _ override exclude specs *add only
    • if excludes begins with | they are additional to global excludes
    • if excludes is empty (/e:) no excludes are used
    • excludes have the form  filespec [| filespec [| filepec] ...]
  • f: folder _ use new add root or extract base
  • s force recursion *add only
  • s- stops recursion *add only
  • <password encrypt with password [between < and end of line so must be last on line]

It is also possible to override the AddRecurseDirs option of the AddOptions property by adding special characters at the beginning of any entry:

  • |  stops recursing folders *add only
  • > forces recursion *add only

NOTE: This is only possible while Adding to a zip archive, When using Extract, Delete or CopyZippedFiles never use this!!!

Further it is possible to override the default password by appending the file specification with a '<' and then the new password or leave it empty to cancel an existing password temporarily as set in the Password property.

NOTE: This is only possible while Adding to or Extracting from a zip archive, When using Delete or CopyZippedFiles never use this!!!

Merge

[zipname] [>>spec] [[ /switch] ...]

zipname a source zip filename
>>spec before any source zip is specified sets the default select spec
>>spec select files in current zip according to spec
 /switch optional switch of form /char:value (note / must follow space)
  /char can be lower case or uppercase
E:[|][spec[|spec]...] excludes _ override exclude specs
  if value begins with | they are additional to global excludes otherwise use spec,
  changes to excludes occur at current line and continue until changed,
  does not change already included files.
  when used on same line as '>>' it only applies to that line and modifies the 'current' excludes.
F: folder _ use new add root or extract base
N[+ or -] no rename with AddNewName (default +)
X:[old]::[new] translate old to new
